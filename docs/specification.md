\## 📰 討論型ニュース分析システム 仕様書



---



\### 1. システムの目的



単一のニュース記事（テキスト）に対し、\*\*楽観的（機会）\*\*、\*\*悲観的（リスク）\*\*、\*\*客観的（ファクト）\*\*の3つの視点から分析と議論（討論）を実施し、そのプロセスを通じて、バイアスの少ない、多角的な最終レポートを自動生成する。



\### 2. システム構成要素（エージェントと役割）



| エージェント名 | 役割（ロール） | 視点と目標 |

| :--- | :--- | :--- |

| \*\*オーケストレーションエージェント\*\* | 司会/進行/合流 | 記事取得→主張→ファクトチェック→反論→レポート作成の順に各エージェントへ依頼し、全体を進行・合流する（現行実装: `OrchestrationAgent`）。 |
| \*\*リサーチャーエージェント\*\* | 記事提供 | 外部から分析対象の\*\*ニュース記事テキスト\*\*を取得し、後続処理へ渡す。 |

| \*\*楽観的アナリスト (A)\*\* | ポジティブ解釈 / 主張者 | 記事内の情報から\*\*メリット、成長、機会\*\*など、前向きな要素を抽出し、その主張を最もらしく提示する。 |

| \*\*悲観的アナリスト (B)\*\* | ネガティブ解釈 / 主張者 | 記事内の情報から\*\*リスク、コスト、課題\*\*など、否定的な要素を抽出し、その主張を最もらしく提示する。 |

| \*\*客観的ファクトチェッカー (C)\*\* | 事実検証 / 議論の監視役 | 議論の過程でAとBが引用した\*\*事実、データ、論拠の正確性や文脈からの逸脱\*\*を中立的にチェックする。 |

| \*\*レポートエージェント\*\* | 最終レポート作成 / 統合者 | A、B、Cの\*\*全フェーズ出力（主張・批評・反論）\*\*を統合し、バイアスを指摘した上でバランスの取れた最終レポートを作成する。 |



\### 3. 処理フロー（討論のフェーズ）



システムは以下の5つの段階を経て最終レポートを作成します。



| フェーズ | 実行エージェント | 実行内容 | 出力 / 次フェーズへの入力 |

| :--- | :--- | :--- | :--- |

| \*\*フェーズ 0\*\* | \*\*オーケストレーション → リサーチャー\*\* | 外部ニュース記事（テキスト）を取得。RSS一致なし等は早期終了（halt）する。 | 記事テキスト / または halt理由。 |

| \*\*フェーズ 1\*\* | \*\*オーケストレーション → A \& B\*\* | 記事テキストに基づき、それぞれの\*\*視点に特化した一次主張（結論）\*\*と、それを裏付ける記事からの\*\*引用（証拠）\*\*を生成。 | Aの主張（結論＋引用）とBの主張（結論＋引用）。 |

| \*\*フェーズ 2\*\* | \*\*オーケストレーション → C (ファクトチェッカー)\*\* | AとBの主張を検証。特に\*\*引用部分が記事の文脈に合っているか\*\*、\*\*誇張がないか\*\*をチェックし、\*\*バイアス指摘\*\*を生成。 | AとBへの批評（検証結果）。 |

| \*\*フェーズ 3\*\* | \*\*オーケストレーション → A \& B\*\* | 相手の一次主張とCの検証結果（批評）を受け、\*\*相手の解釈の弱点や矛盾点を突く\*\*「反論」を生成し、議論を深化させる。 | Aの反論とBの反論。 |

| \*\*フェーズ 4\*\* | \*\*オーケストレーション → レポートエージェント\*\* | \*\*全フェーズの出力（主張、批評、反論）\*\*を統合し、最終レポートを作成する。 | 最終レポート。 |



\### 4. 出力仕様（最終レポートの構成）



レポートエージェントが出力する最終レポートは、以下の要素で構成されます。



1\.  \*\*分析対象記事情報\*\*: 記事タイトル、ソース、要約。

2\.  \*\*一次主張（意見の対立）\*\*:

&nbsp;   \* 楽観的アナリスト（A）の結論と主要論拠

&nbsp;   \* 悲観的アナリスト（B）の結論と主要論拠

3\.  \*\*議論の過程\*\*:

&nbsp;   \* ファクトチェッカー（C）による指摘事項（A, Bの主張における事実の誇張や偏り）

&nbsp;   \* AとBによる相互反論の要点

4\.  \*\*最終統合結論\*\*:

&nbsp;   \* 議論を経て特定された\*\*最も確実な事実\*\*（ファクトチェッカーの検証に基づく）。

&nbsp;   \* 記事が持つ\*\*潜在的な機会（メリット）\*\*と\*\*潜在的なリスク（懸念）\*\*をバランスよくまとめた結論。

&nbsp;   \* （オプション）この結論に至るまでの最も説得力があった議論のポイント。



\### 5. 評価ポイント（システムの検証基準）



このシステムを評価する際は、以下の点に着目します。



\* \*\*討論の品質\*\*: エージェントAとBが単に反対意見を述べるだけでなく、\*\*相手の論拠に基づいた具体的な反論\*\*を生成できているか。

\* \*\*ファクトチェックの機能性\*\*: Cが単なる要約ではなく、\*\*事実の正確性やバイアス\*\*を効果的に指摘できているか。

\* \*\*最終結論のバランス\*\*: レポートエージェントが、\*\*議論の対立点を解消し、多角的な視点を統合した結論\*\*を導き出せているか（単なる折衷案になっていないか）。
